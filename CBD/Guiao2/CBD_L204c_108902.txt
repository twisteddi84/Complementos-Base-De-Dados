Uma grande diferença entre o MongoDB e o Redis é que o MongoDB armazena tudo em formato JSON enquanto que no Redis pode ser armazenado em listas, sets, hashmaps, conjuntos... O Redis armazena as informações em memória que tem uma expiração em cache. O MongoDB armazena em bancos de dados. Um pró do MongoDB para este problema é que se tiver imensos utilizadores o MongoDB é muito escalável horizontalmente e com a quantidade de informação que é preciso ser armazenada neste exercício era fácil de lidar com os JSON. Já no Redis como podemos utilizar vários tipos de armazenamento de dados como listas e sets é mais de lidar com estes problemas.
No Redis os tempos de leitura/escrita são muito rápidos devido ao armazenamento em memória e o tempo de resposta é constante independentemente do tamanho do conjunto de dados. No MongoDB  o tempo de resposta pode variar consoante o tamanho da coleção e da complexidade da pesquisa. Na minha opinião o Redis é mais eficiente para este problema pois como o problema trabalha à volta de um intervalo de tempo é importante a velocidade de execução.
